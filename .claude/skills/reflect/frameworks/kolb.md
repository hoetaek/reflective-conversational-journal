# Kolb 경험학습 사이클

> Kolb(1984)의 경험학습 이론. 경험을 순환적 학습 과정으로 전환한다.
> USER.md에서 정의한 "자기 실험 노트" — 데이터 수집 → 가설 → 검증 → 결론 → 새 가설 사이클.

## 적합한 상황

- 같은 패턴의 실수가 반복될 때 (단순 오답노트를 넘어 가설-검증이 필요)
- 새로운 방법/도구를 시도한 후 효과를 평가할 때
- 행동을 바꿔봤는데 결과가 어떤지 확인할 때
- **"자기 실험"이 필요한 상황** — What-So What-Now What보다 깊은 분석이 필요할 때

## 구조

| 단계 | 질문 | 초점 |
|------|------|------|
| 구체적 경험 (CE) | 무슨 일이 있었나? | 사실 기술 |
| 성찰적 관찰 (RO) | 왜 그런 결과가 나왔을까? | 패턴/원인 분석 |
| 추상적 개념화 (AC) | 어떤 가설/원칙을 세울 수 있나? | 일반화 |
| 능동적 실험 (AE) | 다음에 어떻게 검증할 건가? | 실험 설계 |

### 구체적 경험 (CE)
- 무슨 일이 있었는지 사실만 기술
- 시간, 상황, 행동, 결과

### 성찰적 관찰 (RO)
- 왜 그렇게 됐을까?
- 이전에도 비슷한 경험이 있었나?
- 어떤 패턴이 보이나?

### 추상적 개념화 (AC)
- 이 경험에서 어떤 가설/원칙을 세울 수 있나?
- "~하면 ~가 된다" 형태로 정리

### 능동적 실험 (AE)
- 이 가설을 어떻게 검증할 건가?
- 구체적인 다음 실험 설계
- 성공/실패 기준은?

## 저널 기록 형식

```markdown
**[항목명]**

- `🔬 구체적 경험 (CE):` [무슨 일이 있었나]
- `🔭 성찰적 관찰 (RO):` [왜 그런 결과가 나왔을까? 패턴은?]
- `💡 추상적 개념화 (AC):` [가설/원칙 — "~하면 ~가 된다"]
- `🧪 능동적 실험 (AE):` [다음 실험 — 뭘 해볼 건가? 성공 기준은?]
```

## 유도 질문

- "이런 일이 전에도 있었어?"
- "왜 그런 결과가 나온 것 같아?"
- "이걸 일반화하면 어떤 원칙이 될까?"
- "다음에 한번 시도해볼 만한 게 뭐가 있을까?"
- "어떻게 되면 성공이라고 볼 수 있어?"

## 원칙

- What-So What-Now What보다 **깊이**가 필요할 때 사용
- 가설은 검증 가능해야 함 — "더 열심히 한다"는 가설이 아님
- 능동적 실험은 구체적이고 측정 가능해야 함
- 이전 Kolb 사이클의 AE(실험)가 새 CE(경험)로 이어지는 순환 구조
