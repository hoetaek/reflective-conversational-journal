# Gibbs 성찰 사이클

> Gibbs(1988)의 성찰 사이클. 6단계로 **감정**을 명시적으로 다루며, 경험을 구조적으로 분석한다.

## 적합한 상황

- 감정이 강하게 작용한 경험 (짜증, 좌절, 기쁨, 뿌듯함 등)
- 대인관계에서 일어난 일 (팀원, 상사, 친구와의 상호작용)
- "왜 그때 그런 기분이었을까?" 가 핵심 질문인 상황
- What-So What-Now What에서 감정 부분이 빈약할 때 대안으로

## 구조

| 단계 | 질문 | 초점 |
|------|------|------|
| 기술 (Description) | 무슨 일이 있었나? | 사실 기술 |
| 감정 (Feelings) | 그때 어떤 느낌이었나? | 감정 인식 |
| 평가 (Evaluation) | 뭐가 잘 됐고 뭐가 안 됐나? | 좋았던 점/아쉬운 점 |
| 분석 (Analysis) | 왜 그렇게 됐을까? | 원인 분석 |
| 결론 (Conclusion) | 뭘 배웠나? | 교훈 도출 |
| 행동 계획 (Action Plan) | 다음에 어떻게 할 건가? | 구체적 다음 행동 |

### 기술 (Description)
- 무슨 일이 있었는지 사실만 기술
- 누가 관련됐고, 무슨 말/행동이 오갔는지

### 감정 (Feelings)
- 그 상황에서 어떤 감정을 느꼈나?
- 감정이 중간에 바뀌었나? (처음엔 → 나중엔)
- 상대방은 어떤 감정이었을 것 같나?

### 평가 (Evaluation)
- 이 경험에서 잘 된 부분은?
- 잘 안 된 부분은?

### 분석 (Analysis)
- 왜 잘 됐는지 / 왜 안 됐는지
- 다르게 했으면 어땠을까?

### 결론 (Conclusion)
- 이 경험에서 배운 점
- 일반적으로 적용할 수 있는 교훈

### 행동 계획 (Action Plan)
- 비슷한 상황이 다시 오면 구체적으로 뭘 할 건가?

## 저널 기록 형식

```markdown
**[항목명]**

- `📝 기술:` [무슨 일이 있었나]
- `💭 감정:` [그때 어떤 느낌이었나]
- `⚖️ 평가:` 잘된 점 — [좋았던 점] / 아쉬운 점 — [아쉬운 점]
- `🔍 분석:` [왜 그렇게 됐을까]
- `💡 결론:` [배운 점]
- `✨ 행동 계획:` [다음에 어떻게 할 건가]
```

## 유도 질문

- "그때 기분이 어땠어?"
- "상대방은 어떤 마음이었을 것 같아?"
- "뭐가 잘 됐다고 느꼈어? 반대로 아쉬운 건?"
- "왜 그런 결과가 나온 것 같아?"
- "다음에 비슷한 상황이 오면 어떻게 하고 싶어?"

## 원칙

- **감정이 핵심** — 감정 단계를 건너뛰지 않는다
- 감정에 좋고 나쁨은 없다 — 있는 그대로 인식
- What-So What-Now What보다 **감정에 초점이 필요할 때** 사용
- 모든 6단계를 채울 필요 없음 — 의미 있는 것만
